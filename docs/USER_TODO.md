2. **（iPhone/iPadのみ・任意）検索ページ→共有で本棚追加をiOSでも使うためのショートカット作成**
   iOS SafariはWeb Share Target API（PWAの共有シート連携）に2026-07時点で対応していないため、Androidと同じようにOS標準の共有シートへ直接NovelShelfを出すことができません（[DECISIONS.md](DECISIONS.md)参照）。同等のことをするには、一度だけiOSの「ショートカット」アプリで以下を作成してください:
   1. ショートカットを新規作成し、最初のアクションを「共有シート入力を受け取る」にする
   2. 「URLを開く」アクションを追加し、`https://novelshelf.jp/share?url=[共有シート入力]`を指定する（`[共有シート入力]`の部分は前のアクションの出力を差し込む）
   3. ショートカットの設定で「共有シートに表示」をオンにする
   4. これで、なろう・カクヨム・ハーメルンの作品ページをSafariで開いた状態で共有ボタン→作成したショートカットを選ぶと、NovelShelfに追加されます

3. **なろうの話本文取得方式（HTML取得）についての確認**
   なろう小説APIは作品メタデータの検索のみを提供しており、話一覧・本文は提供していないことが実装時に判明しました。そのため、話一覧・本文は`https://ncode.syosetu.com/`のページから取得する方式にしています（`robots.txt`のCrawl-delay: 1に従い1秒間隔を必ず空ける）。通常は読んでいる話を1話ずつ取得しますが、ユーザーが明示的に「全話をオフライン保存」を押した場合のみ、レート制限を維持したまま全話を順次取得します（自動での一括先読みはしません）。この方針で問題ないか確認をお願いします。懸念があれば、なろう二次利用ガイドラインを一緒に確認します。詳細は[DECISIONS.md](DECISIONS.md)の該当項目を参照してください。

4. **（任意・後回し可）パスワードリセット機能を実際に使うためのSMTP接続情報の設定**
   実装完了（2026-07-19、[DECISIONS.md](DECISIONS.md)参照）。プロバイダを問わずSMTP接続情報を`.env`に設定するだけで動きます。Gmail SMTP（アプリパスワード、個人利用なら一番手軽）/ SendGrid・AWS SES等の専用サービス / ConoHaのメールサービスのいずれかを選び、VPS上の`.env`に`MAIL_HOST`/`MAIL_PORT`/`MAIL_USERNAME`/`MAIL_PASSWORD`/`MAIL_FROM`を設定した上で`docker compose ... up -d`を再実行してください（[DEPLOY.md](DEPLOY.md)ステップ7と同じ手順）。未設定のままでもアプリ自体は問題なく動きます（パスワードリセットメールが届かないだけです）。
