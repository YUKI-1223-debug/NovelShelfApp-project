# NovelShelf 要件定義書

Phase1 成果物。この文書はレビュー対象であり、承認後にPhase2（Docker環境構築）へ進む。

## 1. プロジェクト概要

個人利用専用・広告なしのWeb小説リーダー「NovelShelf」。Android・iPhone・iPad・PCブラウザで動作するPWAとして提供し、読書位置・本棚・設定・しおりをクラウド同期する。市販の電子書籍リーダー（Kindle、Apple Books）に匹敵する快適な読書体験を目標とする。

- 利用者: 開発者本人（個人利用）。マルチユーザー機能（他者との共有等）は対象外。
- 収益化: なし。広告表示なし。
- 対応デバイス: Android / iPhone / iPad / PCブラウザ（レスポンシブ + PWA）

## 2. 対応サービスとデータ取得方針

| サイト | 公式API/フィード | MVP時点の方針 |
|---|---|---|
| 小説家になろう | なろう小説API（公式提供、二次利用ガイドラインあり） | Phase3で最初に実装。作品メタデータ・話一覧取得に利用。本文取得は規約準拠の範囲で実装。 |
| カクヨム | 公式の外部開発者向けAPIなし | サイト個別調査のうえ実装。robots.txt・利用規約を確認し、許可される範囲（自分のブラウザ操作を代行する範囲、公開ページの通常アクセス等）に限定。規約上不可なら本文取得は行わずURL登録・外部リンク遷移のみに縮退。 |
| ハーメルン | 公式の外部開発者向けAPIなし | 同上。個別調査してから実装方針を確定する。 |
| pixiv小説 | pixivはAPI提供あり（画像/イラスト中心、小説领域の外部提供状況は要確認） | 同上。pixiv API利用規約を確認し、許容範囲内でのみ実装。 |

**方針（ユーザー確定事項）**: 4サイトすべてに対応するが、取得方式はサイトごとに個別調査し、実装可能と確認できたものから順次対応する。規約上グレー/不可と判明したサイトは、自動取得を行わず「外部リンクを本棚に登録し、閲覧はブラウザに委譲する」フォールバックモードに縮退する。スクレイピング・保存・再配布を前提とした実装は行わない（詳細は[DECISIONS.md](../DECISIONS.md)）。

## 3. 機能要件

### 3.1 認証
- メールアドレス + パスワードによるログイン/サインアップ
- 将来的にGoogleログインを追加できるよう、認証方式を抽象化した設計とする（例: 認証プロバイダテーブルを分離）
- 個人利用のためメール送信基盤（パスワードリセット等）の要否は要確認（→ USER_TODO）

### 3.2 本棚
- 分類: 読書中 / お気に入り / 読了 / あとで読む / 更新あり
- グルーピング表示: 作者ごと / シリーズごと / サイトごと / タグごと
- 表示順序はユーザー設定として同期

### 3.3 読書画面
- 縦書き / 横書き切り替え
- ダークモード / ライトモード
- 明朝体 / ゴシック体切り替え
- フォントサイズ変更、行間変更、余白変更、背景色変更
- ページ送りモード / スクロールモード
- フルスクリーンモード
- 上記設定はすべてクラウド同期

### 3.4 読書位置同期
- 同期対象: 読んだ話、スクロール位置（またはページ位置）、最終読書日時
- Android・iPhone・PC間でリアルタイムに近い形で同期（デバイス切り替え時に前回位置から再開）

### 3.5 オフライン読書（キャッシュ）
- 各サイトの利用規約・提供APIで許可される範囲でのみキャッシュを実施
- 閲覧済み話は自動キャッシュ、任意作品は明示操作でオフライン保存
- 保存容量表示、キャッシュ削除（作品単位）、保存容量上限設定、差分更新
- オフライン中も読書・しおり作成・読書位置更新が可能。オンライン復帰時に自動同期
- 保存データはアプリ専用ストレージに暗号化して保存。EPUB等へのエクスポート、他サービスへの共有は不可とする

### 3.6 検索
- タイトル / 作者 / ジャンル / サイト / タグによる検索

### 3.7 更新管理
- 更新確認、更新一覧、更新履歴
- 将来的にPush通知に対応できる設計とする（MVPでは非対応、通知トリガーの仕組みだけ設計に組み込む）

### 3.8 しおり
- 複数作成可能。名前・メモ・タグを保存
- タグは自由入力（例: 異世界、恋愛、完結済み、神作品、泣ける、あとで読む）

### 3.9 作者ページ
- 作者ごとの全作品、読了数、お気に入り、更新作品を表示

### 3.10 シリーズ管理
- シリーズ作品をまとめて表示

### 3.11 読書履歴
- 最近読んだ作品、最近読んだ話、読書日時の記録

### 3.12 読書カレンダー
- カレンダー表示、読書時間集計（今日/今週/今月/年間/合計）

### 3.13 読書統計
- 総読了作品数、総読了話数、総読書時間
- 月別読書時間、サイト別読書時間、作者別読書時間

### 3.14 設定同期
- 同期対象: ダークモード、縦横表示、フォント、フォントサイズ、行間、背景色、テーマ、本棚表示順

### 3.15 UI/UX方針
- Kindle・Apple Books・Google Play Books相当のシンプルなUI。装飾を排し読書に集中できる構成
- 広告は一切表示しない
- Material Design 3をベースとする

## 4. 非機能要件

- Docker Composeで一括起動できること
- レスポンシブ対応（スマホ/タブレット/PC）
- PWA対応（インストール可能、オフライン起動）
- 高速表示（初期表示・ページ送りの体感速度を重視）
- クリーンアーキテクチャに基づくバックエンド設計
- OpenAPIによるAPI仕様の型安全な共有（フロントの型はOpenAPIから生成）
- JUnitによる単体テスト、E2Eテスト（Playwright等）
- CI/CD対応（GitHub Actions等を想定、Phase6で具体化）
- SOLID原則・DRY原則・Clean Codeに準拠し、可読性を最優先。コメントは必要最小限

## 5. 技術スタック

- **Frontend**: Next.js, TypeScript, Tailwind CSS, PWA (next-pwa もしくは Serwist)
- **Backend**: Spring Boot, Java 21, PostgreSQL
- **Infrastructure**: Docker, Docker Compose（将来ConoHa VPSへそのままデプロイ可能な構成）

## 6. スコープ外（将来拡張として設計上の余地のみ確保）

- AIによる「前回までのあらすじ」生成
- AIおすすめ作品、AIタグ自動生成、AI読書分析
- 更新のPush通知
- 多言語対応
- テーマ追加
- 読書ランキング
- ホーム画面ウィジェット
- Apple Watch・Wear OS連携
- Googleログイン（設計上は最初から拡張可能にしておく）

## 7. 法的・規約上の制約と対応方針

このアプリは個人利用専用であり、公開・再配布は行わない前提。ただし以下を厳守する。

1. 各サイトの利用規約・robots.txt・提供APIの範囲を超えたデータ取得は行わない
2. 規約上の位置づけが不明なサイトについては、実装前に利用規約を確認し、[USER_TODO.md](../USER_TODO.md) に確認依頼を記載する
3. 取得したコンテンツの他サービスへの共有・エクスポート・再配布機能は実装しない
4. キャッシュデータは暗号化し、アプリ専用ストレージ以外に出さない

## 8. 未決事項・要確認事項

- カクヨム/ハーメルン/pixiv小説の具体的な取得可否判定（Phase3着手前に個別調査が必要）
- パスワードリセット等でのメール送信基盤の要否（SMTP契約の要否）
- ConoHa VPSの具体的なプラン・スペック（Phase6デプロイ時に確定でよい）
- 実際にアプリで使うオフラインキャッシュの暗号化方式（端末内DB暗号化 or ファイル暗号化）の詳細技術選定はPhase4/5で決定
